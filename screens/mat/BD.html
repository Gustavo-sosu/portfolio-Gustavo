Sobre a Matéria

Professor: Alexandre Bernardes

O que é?

Técnicas avançadas em bancos de dados relacionais e não relacionais referem-se a estratégias e ferramentas mais complexas utilizadas para otimizar o desempenho, escalabilidade e segurança desses sistemas de gestão de dados. Em bancos de dados relacionais, estas técnicas podem incluir a criação de índices avançados, otimização de consultas, partições e replicação.

Objetivos

Durante as aulas tivemos como objetivo principal compreender as diversas questões logicas, técnicas e praticas sobre situações que são enfrentadas no mercado de trabalho.

Os componentes mais basicos da matéria foram aprendidos durantes os anos anteriores. Como por exemplo:

- Extração de Dados de uma Tabela (SELECT, FROM e WHERE)
- Manipulação de Dados Numéricos (+, -, *, /, %)
- Consultas Qualificadas (AND, OR, NOT e =, >=, <>, <=, >, <)
- Operadores Avançados (BETWEEN, IN, LIKE, IS NULL)
- Agrupamento de Dados (GROUP BY, HAVING)
- Funções de Agregação (COUNT, SUM, AVG, MAX, MIN)
entre outros.

View

É uma tabela virtual que armazena a definição de uma consulta SQL. Ela funciona como uma tabela real, mas não contém dados fisicamente armazenados.
Ao invés disso, os dados são gerados dinamicamente quando a view é consultada, baseando-se na query que a define.




O que são Procedures e Functions?


Quando uma aplicação executa uma query SQL comum, o servidor precisa interpretá-la, compilá-la e executá-la a cada vez, o que pode causar lentidão em ambientes com muitos dados ou alta concorrência. Para melhorar o desempenho, reutilizar código e organizar melhor as regras de negócio, os SGBDs (como PostgreSQL, Oracle e SQL Server) permitem encapsular comandos SQL em blocos reutilizáveis chamados Stored Procedures e Functions. Esses blocos são armazenados no servidor e executados localmente, o que reduz o tráfego de rede, aumenta a segurança e melhora o tempo de resposta das aplicações.


Diferença entre Function e Procedure

Característica Function Procedure
Retorno Sempre retorna um valor Pode retornar ou não
Uso mais comum Cálculos, consultas, transformações Ações como inserção, exclusão etc
Suporte a múltiplos resultados Sim (inclusive com RETURNS TABLE) Não diretamente, mas pode usar parâmetros OUT

Tipos de Funções no PostgreSQL



Funções em Linguagem SQL

• Utilizam apenas comandos SQL simples: SELECT, INSERT, UPDATE, DELETE.
• Não suportam estruturas de controle (como IF, LOOP).
• Indicadas para operações simples e diretas.


Funções Procedurais (PL/pgSQL)

• Suportam variáveis, loops, condições, cursores etc.
• Usam a linguagem PL/pgSQL, semelhante ao PL/SQL do Oracle.
• Podem ser utilizadas para lógicas de negócio complexas.
• Outras linguagens suportadas incluem: PL/Python, PL/Perl, PL/Tcl.


Funções em Linguagens Externas

• Criadas com linguagens como C, Java, Python.
• Permitem a criação de algoritmos complexos e de alto desempenho.
• Exigem instalação e configuração adicionais no servidor PostgreSQL.

Papel:

Um papel (role) é uma entidade no PostgreSQL utilizada para gerenciar permissões e acessos ao banco de dados. Ela pode ser configurada para realizar login (atuando como um usuário), agrupar outros papéis (funcionando como um grupo) ou exercer ambas as funções. Além disso, papéis podem ser proprietários de objetos no banco e atribuir ou herdar privilégios de outros papéis.

Comandos:

CREATE ROLE nome [ [ WITH ] opção [ ... ] ]
Abaixo estão as opções que podem ser utilizadas para criar um papel:

Opção Descrição
LOGIN / NOLOGIN Permite ou não o login.
SUPERUSER / NOSUPERUSER Define se é superusuário
CREATEDB / NOCREATEDB Pode ou não criar bancos
CREATEROLE / NOCREATEROLE Pode ou não criar papéis
INHERIT / NOINHERIT Herda ou não privilégios de papéis aos quais pertence
ENCRYPTED / UNENCRYPTED Define se a senha é criptografada
VALID UNTIL 'YYYY-MM-DD' Expira após essa data
CONNECTION LIMIT n Limita conexões simultâneas (-1 = sem limite)
IN ROLE, ROLE Define relacionamentos com outros papéis

alter role nome_ papel with encrypted password ‘123’ -- Para Alterar um papel (role)
drop role nome_papel -- exclui o papel
Select * from pg_user -- Para Listar todos os papéis


Grupos:

Grupos são representados por papéis sem permissão de login.

Comandos:

create role nome_grupo -- Criar um papel - grupo
Grant nome_grupo to user1, user2, user3 -- Adiciona usuários a um grupo
Revoke nome_grupo from user3 -- Remove usuários de um grupo
drop role nome_grupo -- Para Excluir um papel - grupo
Select * from pg_group -- Para Listar todos os grupos

Privilegios:

Privilégios são permissões que determinam o que um usuário pode ou não pode fazer dentro do banco de dados.

Superusuário:
Possui acesso completo e irrestrito a todos os objetos e operações em qualquer banco de dados do PostgreSQL.

Propriedade de Objetos:
Quando um usuário cria um objeto (como um banco de dados, tabela, view, função etc.), ele se torna automaticamente o proprietário desse objeto.

Concessão de Privilégios:
Para que outros papéis ou usuários possam acessar ou manipular um objeto, é necessário conceder explicitamente os privilégios apropriados (como SELECT, INSERT, UPDATE, DELETE, entre outros).

Comandos:

Grant update on nome_tabela to nome_papel -- Dando privilégios a um papel
Grant select on nome_tabela to nome_grupo -- Dando privilégios a um papel
Revoke all on nome_tabela from public -- Removendo todos os privilégios de todos os usuários


Apps utilizados

pgAdmin

O pgAdmin é uma ferramenta de código aberto usada para gerenciar e desenvolver bancos de dados PostgreSQL. É uma interface gráfica que permite aos usuários interagir com o banco de dados, executando consultas SQL, visualizando e modificando estruturas de dados e monitorando o desempenho.
