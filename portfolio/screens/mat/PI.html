O que é?

O Projeto Integrador I é uma disciplina que permite aos estudantes colocar em prática os conhecimentos adquiridos ao longo do curso, por meio do desenvolvimento de sistemas voltados para clientes reais.
Nela, os alunos enfrentam desafios concretos do mercado, ganhando experiência prática e aproximando-se do ambiente profissional.

Objetivos

O objetivo dessa matéria é ajudar os alunos a desenvolverem habilidades práticas desenvolvendo seus própios projetos, além de proporcionar uma experiência prática que simula o ambiente de trabalho real.

Conteudo

Durante todo o Semestre, vimos diversos conteudos e agora por meio deste projeto venho falar um pouco sobre o que estudei ao longo desse semestre.




O que é o Git?

O Git é um sistema de controle de versão distribuído, usado principalmente no desenvolvimento de software. Ele permite que várias pessoas trabalhem no mesmo projeto ao mesmo tempo, mantendo um histórico completo de todas as alterações feitas nos arquivos.

O que foi ensinado?

Durante as aulas, aprendemos sobre como implementar o Git em nossos projetos, como criar repositórios, fazer commits, branches e pull requests utilizando o GitHub como plataforma de hospedagem.
E diversos comandos do Git.

Comandos do Git:

git config --list = Exibe algumas informações e o nome e email de quem está logado no PC (caso tenha alguém logado). Em casos de erro, aperte "q".
git config --global user.name "Nome do Usuário" = Informa o nome do usuário que deseja logar.
git config --global user.email "Email do usuário" = Informa o email do usuário.
git config --global --unset-all user.name = Limpa os nomes logados.
git config --global --unset-all user.email = Limpa os emails logados.
git init = Cria um repositório na pasta (repositório oculto).
git branch -m master main = Muda o nome da branch de master para main.
git status = Mostra o que tem na pasta.
git add <Nome do arquivo> = Adiciona um arquivo ao diretório.
git add -all ou git add . = Adiciona todos os arquivos ao diretório.
git commit -m "mensagem do commit" = Salva as alterações e exibe a mensagem dessa alteração.
git log = Mostra os commits (alterações) realizados no diretório.
git restore --staged <arquivo> = Volta um arquivo específico.
git restore --staged . = Volta todos os arquivos.
git restore [nome ou .] = Retorna o diretório antigo (um arquivo ou todos).
git reset --hard = Reseta tudo para o último commit confirmado.
git diff = Verifica o que foi alterado.
git log -p, git log --oneline, git log --graph = Mostra as alterações adicionadas.
git checkout = Volta para qualquer outro commit (save).
git reset HEAD~1 = Desfaz a confirmação do último save.
git reset --hard = Apaga o último save.
git reset --soft = Desfaz o último commit, mas mantém as modificações em aberto.
git revert [código do save (oneline)] = Reverte um commit específico.
git --abort = Aborta o processo de revert.
Ao entrar no revert: Aperte Insert para editar, ALT+Q para sair do modo edição, :wq para confirmar e sair.
.gitignore = Arquivo para ignorar alterações em determinados arquivos/pastas.
git push origin main = Envia alterações para o repositório remoto.
git pull origin main = Puxa alterações do repositório remoto.
git branch = Checa as branches atuais.
git checkout nome_branch = Troca de branch.
git branch -d nome_branch ou git branch -D nome_branch = Apaga a branch localmente.
git push --delete origin nome_branch = Apaga a branch no GitHub.
git merge nome_branch = Mescla a branch atual com a branch especificada.

Apps utilizados

GitHub

O GitHub é uma plataforma online de hospedagem de código-fonte que usa o Git como base. Ele foi lançado em 2008 e é hoje uma das maiores comunidades de desenvolvedores do mundo. O GitHub permite que você envie (ou "publique") seus projetos que estão versionados com Git para a internet, facilitando o compartilhamento e a colaboração com outras pessoas.

Em termos simples:
O GitHub é como uma “rede social de código”. Ele armazena os repositórios (projetos Git) na nuvem e oferece diversas ferramentas para colaboração, gerenciamento de projetos e integração com outras plataformas.

Git

O Git é um sistema de controle de versão distribuído criado por Linus Torvalds, o mesmo criador do Linux, em 2005. Ele foi desenvolvido para ajudar equipes de desenvolvimento a gerenciar o código-fonte de seus projetos de forma eficiente, segura e colaborativa.

Em termos simples:
O Git permite que você acompanhe todas as modificações feitas em um projeto, mantenha um histórico completo das mudanças e colabore com outras pessoas, mesmo que todos estejam trabalhando em partes diferentes do projeto ao mesmo tempo.
