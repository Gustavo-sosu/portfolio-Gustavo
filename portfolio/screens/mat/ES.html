Sobre a Matéria

O que é?

A engenharia de software é uma disciplina que se preocupa com os aspectos da produção do software, desde sua concepção inicial até sua operação e manutenção.

Objetivos

O objetivo da disciplina Engenharia de Software é capacitar o aluno a entender, planejar, desenvolver, testar, manter e gerenciar sistemas de software de forma sistemática, eficiente e com alta qualidade. Essa disciplina fornece os fundamentos teóricos e práticos para que o aluno compreenda o ciclo de vida do desenvolvimento de software, desde a análise de requisitos até a entrega e manutenção do produto final.

Conteudo

Durante todo o Semestre, vimos diversos conteudos e agora por meio deste projeto venho falar um pouco sobre o que estudei ao longo desse semestre.




Processo Unificado (RUP)

RUP (Rational Unified Process): Processo iterativo e incremental orientado a objetos, centrado em casos de uso, arquitetura e risco.

Fases: Concepção, Elaboração, Construção e Transição.

Fluxos de trabalho:

  • Engenharia: modelagem de negócios, requisitos, análise/desenho, implementação, testes, implantação.
  • Incremental: desenvolvimento por versões com feedback contínuo.
  • Apoio: gerenciamento de mudanças, projetos e ambiente.

Engenharia de Requisitos

Engenharia de Requisitos: Processo de descobrir, analisar, documentar e validar requisitos.

Tipos de Requisitos:

  • Funcionais: serviços que o sistema deve fornecer.
  • Não-funcionais: restrições (tempo, segurança, usabilidade etc.).

Elicitação: entrevistas, casos de uso, cenários, etnografia.

Etapas: descoberta, classificação, priorização, documentação.


UML

Atores representam pessoas, sistemas ou entidades externas que interagem com o sistema. Casos de uso descrevem funcionalidades ou serviços oferecidos pelo sistema do ponto de vista do usuário.

Atores e Casos de Uso:

  • Casos de uso: Descrevem interações entre usuários (atores) e o sistema.
  • Atores: Representam usuários ou sistemas que interagem com o software.
  • Associações: include, extend, generalização.
  • Documentação: fluxo principal, alternativo, exceções, pré e pós-condições.

Modelagem de Sistemas

Criação de representações do sistema sob diferentes perspectivas

Modelagem:

  • Contexto, interação (casos de uso e sequência), estrutura (classe), comportamento (atividades, estados).

Modelos UML usados:

  • Diagramas de caso de uso, sequência, classe, atividades, máquina de estados.

Projeto de Arquitetura

Projeto arquitetural é a organização do sistema em componentes e suas interações. Inclui diferentes visões arquiteturais: lógica, processo, desenvolvimento e física.

Projeto arquitetural: Organização do sistema em componentes.

Visões arquiteturais: lógica, processo, desenvolvimento, física.

Padrões arquitetônicos:

  • MVC, Camadas, Repositório, Cliente-servidor.

Critérios de decisão: desempenho, segurança, disponibilidade, manutenibilidade.


UML: Diagrama de Classe

O Diagrama de Classe é utilizado para representar a estrutura estática de um sistema, mostrando as classes, seus atributos, métodos e os relacionamentos entre elas. Permite visualizar como os objetos se organizam e interagem, facilitando o entendimento e o projeto da arquitetura do software.

  • Relacionamentos: associação (simples, binária, n-ária), agregação, composição, herança, dependência, realizações.
  • Conceitos adicionais: visibilidade, multiplicidade, interfaces, restrições (inclusive com OCL).

Apps utilizados

Astah

Astah é uma ferramenta de modelagem de software que utiliza a Linguagem de Modelagem Unificada (UML) para criar diagramas. Ela é usada para visualizar, projetar e documentar sistemas de software, facilitando a comunicação entre desenvolvedores e outras partes interessadas.
